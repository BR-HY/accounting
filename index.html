Setup

还差 Supabase 配置

把项目 URL 和 anon key 填进配置文件后，这个站点就能真正登录和同步。

请编辑当前目录下的 config.js，填入你自己的 Supabase 项目地址和匿名公钥。匿名公钥可以公开放在前端里，但不要把 service role key 放到这个仓库。

  1. 在 Supabase 新建免费项目。
  2. 运行 supabase/schema.sql 初始化表和权限。
  3. 在 config.js 里填写 supabaseUrl 和 supabaseAnonKey。
  4. 把 GitHub Pages 网址加入 Supabase 的 Site URL 和 Redirect URLs。

Gate

先登录，再开始记账

这样每条记录都会自动和当前账号绑定，天然支持多设备同步和用户隔离。

登录成功后，下面的录入区、账目列表和报表会自动展开。如果你之前在浏览器本地版里记过账， 系统会尝试把旧的本地记录一次性迁移到你的云端账号下。
